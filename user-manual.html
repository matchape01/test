Page d'accueil Lecture
Point d'entrée central vers toutes les pages

La page d'accueil (index.html) est le hub de navigation. Elle permet d'accéder en un clic à l'ensemble des outils du festival.

Navigation disponible

Carte / BoutonDescription
📅 Rapport 1 — Agenda par JourVoir tous les ateliers filtrés par jour ou par type
👤 Rapport 2 — Agenda par PersonneVoir le planning d'une personne spécifique (Fac, Helper, Angel, Traducteur)
📋 Rapport 3 — Logistique par AtelierConsulter la fiche logistique complète d'un atelier
⚙️ AdminÉditer les données d'un atelier et sauvegarder
🗂 Master DataGérer les référentiels (facilitateurs, helpers, lieux, etc.)
ℹ️
Toutes les pages doivent être ouvertes via http://localhost:3000 (serveur local Python). Lancez d'abord start.bat avant d'ouvrir le navigateur.

Ce rapport affiche l'ensemble des ateliers du festival, organisés dans un tableau avec tri et filtres. Il est idéal pour avoir une vue d'ensemble d'une journée ou d'une catégorie d'atelier.

Boutons d'action

🏠 Home 🔄 Refresh 📱 Vue Mobile 🌙 Mode Nuit
  • 🔄 Refresh — recharge la page pour prendre en compte les dernières modifications de data.js
  • 📱 Vue Mobile / 🖥 Vue Tableau — bascule entre un tableau et des cartes groupées par jour (automatique sous 768px)

Rapport 3 — Logistique par Atelier Lecture
Fiche logistique complète d'un atelier

Ce rapport affiche la fiche complète d'un atelier sélectionné : équipe, salle, note et informations logistiques détaillées. C'est la page de référence pour la préparation terrain.

Navigation en 2 étapes

1
Choisir le jour — sélectionnez un jour parmi les 7 disponibles. Le sélecteur d'ateliers s'active automatiquement.
2
Choisir l'atelier — la liste affiche tous les ateliers de ce jour au format [HH:MM] Nom de l'atelier. Sélectionnez-en un pour afficher la fiche.

Contenu de la fiche

ZoneContenu
En-têteNom de l'atelier, heure, jour, date, type et lieu
Facilitateur(s)Badges verts — personnes en charge de l'animation
Helper(s)Badges bleus — personnes en soutien
AngelBadge rose — personne en charge du soutien émotionnel
TraducteurBadge bleu clair — traducteur assigné
NoteNote courte normalisée (ex : "Shift Colibri", "Atelier + long")
Informations LogistiquesTexte libre détaillé pour la préparation terrain
💡
Si les champs Note ou Informations Logistiques affichent un texte en italique grisé, c'est qu'aucune information n'a encore été saisie pour cet atelier. Utilisez Admin pour les compléter.
⚙️
Admin — Édition des ateliers Édition
Modifier les données d'un atelier et sauvegarder dans data.js

La page Admin permet de modifier tous les champs d'un atelier existant et de sauvegarder directement dans data.js via le serveur local.

⚠️
Le serveur local (start.bat) doit être actif pour que la sauvegarde fonctionne. Sans lui, le bouton "Valider" ne peut pas écrire sur le disque.

Sélection de l'atelier à modifier

1
Choisir le jour dans le premier menu déroulant.
2
Choisir l'atelier dans le second menu (se déverrouille automatiquement).
3
Le formulaire s'affiche pré-rempli avec les données actuelles de l'atelier.

Champs du formulaire

ChampTypeSource des options

DateTexte (auto-rempli)Calculé depuis le Jour

Lieu / SalleListe déroulanteref_lieux.js
Type d'atelierListe déroulanteref_types.js
Nom de l'atelierTexte libreSaisie manuelle
Facilitateur 1 & 2Listes déroulantesref_facilitateurs.js
TraducteurListe déroulanteref_traducteurs.js
Helper 1, 2 & 3Listes déroulantesref_helpers.js
AngelListe déroulanteref_angels.js
NoteListe déroulanteref_notes.js
Informations LogistiquesZone de texte libreSaisie manuelle

Sauvegarde

1
Modifiez les champs souhaités dans le formulaire.
2
Cliquez sur ✅ Valider et sauvegarder data.js — une boîte de confirmation s'affiche.
3
Confirmez : data.js est écrit sur le disque via le serveur local. Un message de succès apparaît.
4
Cliquez 🔄 Refresh dans les rapports pour voir les nouvelles données.

Master Data — Gestion des référentiels Admin
Ajouter, renommer ou supprimer des entrées dans les listes de référence

Master Data est la page de gestion des listes de référence utilisées dans Admin et dans les rapports. Chaque modification est sauvegardée dans le fichier ref_*.js correspondant, avec option de mise à jour simultanée de data.js.

Les 7 référentiels disponibles

OngletFichierUtilisé dans
🎓 Facilitateursref_facilitateurs.jsChamps Fac1 / Fac2 dans Admin
🤝 Helpersref_helpers.jsChamps Helper 1/2/3 dans Admin
👼 Angelsref_angels.jsChamp Angel dans Admin
🌐 Traducteursref_traducteurs.jsChamp Traducteur dans Admin
📍 Lieuxref_lieux.jsChamp Lieu / Salle dans Admin
🏷️ Types d'atelierref_types.jsChamp Type d'atelier dans Admin
📝 Notesref_notes.jsChamp Note dans Admin & Rapport 3

Actions disponibles par entrée

✏️ Éditer une valeur

1
Cliquez sur ✏️ à droite de la ligne à modifier.
2
Modifiez le texte dans le champ en ligne (Entrée pour valider, Échap pour annuler).
3
Cliquez ✔ pour confirmer. La barre de sauvegarde apparaît en bas.

+ Ajouter une entrée

1
Cliquez sur le bouton + Ajouter un [type] en bas du tableau.
2
Une nouvelle ligne s'ouvre en mode édition. Saisissez la valeur et validez avec ✔.

🗑 Supprimer une entrée

1
Cliquez sur 🗑 à droite de la ligne. Une boîte de confirmation s'affiche.
2
Confirmez la suppression. La ligne disparaît et la barre de sauvegarde s'affiche.

Sauvegarder les modifications

Après toute modification, une barre orange apparaît en bas de la section avec deux options :

💾 Télécharger ref_[type].js 💾 + data.js mis à jour
BoutonEffetQuand l'utiliser
💾 Télécharger ref_[type].js Sauvegarde uniquement le fichier référentiel Ajout ou suppression d'une entrée sans impact sur les ateliers existants
💾 + data.js mis à jour Sauvegarde le référentiel et met à jour data.js en renommant les occurrences Renommage d'une entrée existante — les ateliers qui la référencent sont mis à jour automatiquement
⚠️
En cas de renommage, utilisez toujours le bouton + data.js mis à jour pour que les ateliers existants reflètent le nouveau nom. Sans cela, les ateliers gardent l'ancienne valeur.
ℹ️
L'ID technique de chaque entrée (ex : F_SABRYNA) ne change jamais — seule la valeur affichée peut être modifiée. Cela garantit la stabilité des références internes.
🗄️
